# Review Skill Proposals

Review proposals generated by the session analysis hook and decide whether to convert them into skills.

## Process

1. **List pending proposals**

Check `~/.claude/skill-proposals/` for pending proposals:

```bash
ls -la ~/.claude/skill-proposals/
```

If no proposals exist, inform the user and exit.

2. **For each proposal, display:**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 Proposal: [filename]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Type: [pattern_type]
Confidence: [confidence]
Source: [session_id]

Pattern Summary:
[pattern_summary]

Suggested Triggers:
- [trigger1]
- [trigger2]
...

Example Workflow (if available):
1. [step1]
2. [step2]
...
```

3. **Ask user for decision**

For each proposal, ask:
"What would you like to do with this proposal?"

Options:
- **Approve**: Convert to skill
- **Reject**: Delete proposal
- **Skip**: Keep for later review
- **Edit**: Modify triggers/name before approving

4. **If Approved:**

Create skill in `~/.claude/skills/[proposed_name]/`:

```markdown
---
name: [proposed_name]
slug: [proposed_name]
scope: shared
owner: session-analysis
last_updated: [today]
status: active
version: 1.0.0
description: [pattern_summary first 100 chars]

triggers:
  - [trigger1]
  - [trigger2]
---

# [Proposed Name]

## Pattern

[pattern_summary]

## When to Use

Use this skill when:
- [derived from triggers]

## Workflow

[example_workflow if available]

## Source

Auto-generated from session: [session_id]
Confidence: [confidence]
```

**Register Skill Triggers (REQUIRED)**

After creating the SKILL.md, MUST add trigger keywords to `~/.claude/skills/skill-rules.json`:

```bash
# Extract triggers from the proposal file
TRIGGERS='[trigger1, trigger2, ...]'  # From "Suggested Triggers" section

# Add entry to skill-rules.json using jq
jq --arg skill "[proposed_name]" --argjson triggers "$TRIGGERS" \
  '.skills[$skill].promptTriggers.keywords = $triggers' \
  ~/.claude/skills/skill-rules.json > /tmp/skill-rules.json.tmp && \
  mv /tmp/skill-rules.json.tmp ~/.claude/skills/skill-rules.json

# Verify the entry was added
jq '.skills."[proposed_name]"' ~/.claude/skills/skill-rules.json
```

This step is **mandatory**. Skills without skill-rules.json entries will NEVER auto-activate.

Then run the skill index rebuilder:
```bash
python3 ~/claude-env/scripts/rebuild-skill-index.py ~/.claude/skills/
```

5. **If Rejected:**

Move proposal to `~/.claude/skill-proposals/rejected/`:
```bash
mkdir -p ~/.claude/skill-proposals/rejected/
mv [proposal_file] ~/.claude/skill-proposals/rejected/
```

6. **Summary**

After processing all proposals, display:
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 Review Summary
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ Approved: [count]
❌ Rejected: [count]
⏭️ Skipped: [count]

New skills created:
- [skill1]
- [skill2]
```

## Notes

- Proposals are in `~/.claude/skill-proposals/`
- Approved skills go to `~/.claude/skills/`
- Rejected proposals move to `~/.claude/skill-proposals/rejected/`
- Rebuild index after creating new skills
- Review regularly to keep skill library current
- **Trigger registration is mandatory. Skills without skill-rules.json entries will never auto-activate.**
